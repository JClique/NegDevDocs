# Proof Read

This skill focuses on improving a single documentation article to make it more accessible and beginner-friendly.

## What I do

I analyse and improve single documentation articles to make them more accessible to beginners. I focus on:

- Breaking down complex concepts into digestible pieces
- Removing unnecessary jargon or explaining it clearly
- Improving paragraph flow and sentence structure
- Reducing cognitive load by chunking information
- Adding helpful transitions between sections

## When to use me

Use this when reviewing or editing a single documentation article. This skill is ideal for:

- Documentation PR reviews
- Pre-publishing checks
- Improving existing guides for new audiences
- Making technical content more beginner-friendly

## Approach

### Readability improvements

- Replace passive voice with active voice
- Use shorter sentences (aim for 15-25 words average)
- Break long paragraphs into shorter ones (3-4 sentences max)
- Replace complex words with simpler alternatives when possible
- Use bullet points for lists of 3+ items

### Reducing information overload

- Identify and extract advanced topics into separate "Advanced" or "Deep dive" sections
- Move optional/edge-case information to collapsible sections or footnotes
- Highlight prerequisites clearly at the start
- Use warnings/callouts sparingly for critical information only
- Remove redundant explanations
- Consolidate duplicate steps

### Beginner-friendly changes

- Add examples for abstract concepts
- Include transitional phrases ("For example", "In contrast", "As a result")
- Define technical terms inline on first use
- Add context for why something is done, not just how
- Use consistent formatting throughout

## How to use

1. Read the full article to understand the overall flow
2. Identify areas where beginners might struggle
3. Apply readability and simplification changes
4. Verify the core message and technical accuracy remain intact
5. Present the improved version with a brief summary of changes

## Limitations

- Focus on one article at a time
- Preserve technical accuracy at all times
- Don't oversimplify to the point of losing important details
- Respect the original author's voice and intent
